What changed from Rev 5

  • Unified design language between audit-ON and audit-OFF (single agent built both files)
  • Same nav switching mechanism (data-nav + switchNav), same audit status indicator markup, same System view layout, same Activity view column order, same drawer function names, same version number
  • System view: SVG-icon header cards, 24h area chart, Pipeline stats + Recent errors list in BOTH files
  • Audit-ON: removed "Opt-out honored" green badge
  • Audit-ON: added "Opted out" badge for tombstoned conversations (these show ETS data only — same card style as audit-OFF cards, with a quiet "Opted out" badge)
  • Audit-ON: Activity row click opens the conversation's slide-over with the relevant message highlighted
  • Audit-OFF: slide-over request row click expands inline to show request details (status, latency, PII breakdown, response size), with a "View in Activity" link to the Activity tab
  • Tombstoned conversation in audit-OFF looks identical to other audit-OFF cards (no badge, no warning)

What to test in Rev 6

  • In audit-ON, the "Opted out" card (card 5) — does the badge make sense? Is it distinct from the audit-OFF card style?
  • In audit-ON, click an Activity row — does the conversation's slide-over open with the relevant message highlighted?
  • In audit-OFF, click a request row in the slide-over — does it expand inline? Is the "View in Activity" link useful?
  • Compare the System view in both files — does it look identical (as intended)?
  • Does the unified design language between the two files feel coherent, or are there remaining inconsistencies?
  • In audit-ON, are tombstoned conversations visually distinct from normal audit-ON conversations? (They should be — different card layout, "Opted out" badge)
  • In audit-OFF, is the tombstoned conversation (card 5) visually identical to other audit-OFF cards? (It should be — no badge)
