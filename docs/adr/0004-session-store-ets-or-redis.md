# Conversation store is ETS (default) or Redis

Conversations are ephemeral by design and do not require a relational database. ETS is the default backend because it is in-process, zero-config, and fast enough for a single-node deployment. Redis is supported via `CONVERSATION_STORE_BACKEND=redis` for deployments that need conversation sharing across multiple nodes or persistence across restarts. No other backends are planned; if durable long-term conversation history becomes a requirement, that would be a new feature rather than an extension of the current conversation store. Audit Mode (SQLite, encrypted, with opt-out) is the durable history mechanism, but it is **not a hot-store** — it does not replace ETS/Redis as the request-path Conversation store. There is no boot loading from SQLite into the Hot Store. See ADR 0010.
