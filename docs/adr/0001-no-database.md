# No database

This application operates as a stateless proxy with no persistent write model for application state. There is no Ecto and no Postgres for application state — conversations are ephemeral (stored in ETS or Redis), and the only durable state lives in the external AI providers themselves. Ecto and SQLite3 are used for Audit Mode only: write-only, encrypted PII columns, no boot loading, no hot-store coupling, supervised by `ShhAi.Application` and stored at `priv/audit/audit.db` (or `AUDIT_DB_PATH`). See ADR 0010. This keeps the deployment footprint small, removes a whole class of operational concerns, and aligns with the project's goal of being a lightweight PII-sanitizing gateway rather than a data-collecting platform.
